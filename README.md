# موقع مسابقة قناة تيليجرام — تتبع الدعوات

موقع كامل: كل مشترك **ينشئ حساباً** باسمه ومعرف تيليجرام وكلمة مرور ⟵ ياخذ رابط دعوة خاص فيه ⟵ يشارك الرابط ⟵ يقدر يرجع يسجّل دخول بأي وقت ليشوف عدد انضماماته الحالي ويعيد نسخ رابطه. وفيه لوحة متصدرين علنية (أفضل 5)، ولوحة تحكم كاملة للمشرف.

## 1) جهّز البوت (5 دقائق)

1. افتح محادثة مع [@BotFather](https://t.me/BotFather) في تيليجرام.
2. أرسل `/newbot` واتبع التعليمات، وخذ **التوكن** (شكله: `123456:ABC-...`).
3. أضف البوت **أدمن** في قناتك، وفعّل له صلاحية **"دعوة مستخدمين عبر رابط" (Invite Users via Link)**.
4. احصل على **آيدي القناة الرقمي**:
   - أرسل أي رسالة بالقناة، ثم حوّلها (Forward) لبوت [@userinfobot](https://t.me/userinfobot) — راح يعطيك رقم يبدأ بـ `-100...`.
   - أو خلي القناة عامة مؤقتاً واستخدم `@اسم_القناة` بدل الرقم.

## 2) ارفع الموقع على استضافة مجانية

أسهل خيار: **Render.com** (يدعم Flask مجاناً ورابط HTTPS جاهز).

الخطوات:
1. أنشئ حساب على [render.com](https://render.com) وربطه بحساب GitHub.
2. ارفع مجلد المشروع هذا إلى مستودع GitHub جديد.
3. من Render: New → Web Service → اختر المستودع.
4. Build Command: `pip install -r requirements.txt`
5. Start Command: `gunicorn app:app`
6. من قسم Environment، أضف المتغيرات التالية:

| المتغير | القيمة |
|---|---|
| `BOT_TOKEN` | توكن البوت من BotFather |
| `CHANNEL_ID` | آيدي القناة (مثال: `-1001234567890`) |
| `ADMIN_PASSWORD` | كلمة مرور تختارها للوحة التحكم |
| `WEBHOOK_SECRET` | كلمة عشوائية سرية (مثال: `x7k9pQ2m`) |
| `SECRET_KEY` | أي نص عشوائي طويل |

7. اضغط Deploy، وانتظر لين يصير الموقع شغال (راح تاخذ رابط مثل `https://your-app.onrender.com`).

## 3) اربط تيليجرام بالموقع (مرة وحدة فقط)

افتح بالمتصفح هذا الرابط (غيّر الدومين والقيم لقيمك):

```
https://your-app.onrender.com/setup-webhook
```

إذا رجع لك `{"ok": true, ...}` معناها ربط ناجح. من هذه اللحظة، أي شخص ينضم للقناة عبر رابط أحد المشتركين، بينحسب تلقائياً.

## 4) الاستخدام

- **صفحة إنشاء الحساب**: `https://your-app.onrender.com/` — شاركها مع جمهورك عشان ينشؤون حساباتهم ويأخذون روابطهم.
- **تسجيل الدخول**: `https://your-app.onrender.com/login` — لأي مشترك يرجع لاحقاً يشوف رابطه وعدد انضماماته (زر "تسجيل الدخول" موجود بأعلى كل صفحة).
- **حساب المشترك**: `https://your-app.onrender.com/account` — يظهر تلقائياً بعد تسجيل الدخول: الرابط مع زر نسخ، وعدد الانضمامات ويتحدّث تلقائياً كل 10 ثوانٍ.
- **لوحة المتصدرين**: `https://your-app.onrender.com/leaderboard` — عامة، تعرض أفضل 5 مشتركين، تتحدّث تلقائياً.
- **لوحة التحكم**: `https://your-app.onrender.com/dashboard` — بكلمة المرور اللي حطيتها بـ `ADMIN_PASSWORD`، تعرض كل المشتركين وتفاصيلهم.

## ملاحظات مهمة

- إذا كان عندك نسخة سابقة من المشروع (بدون حسابات/كلمات مرور) شغّالة فعلياً، **احذف ملف `contest.db` القديم** قبل رفع هذه النسخة، لأن مخطط قاعدة البيانات تغيّر (أضيف عمود كلمة المرور وأصبح معرف تيليجرام إلزامياً وفريداً لكل حساب).
- خطة Render المجانية تنام بعد فترة خمول وتحتاج ثواني للاستيقاظ عند أول زيارة — طبيعي.
- قاعدة البيانات SQLite ملف محلي؛ في الخطة المجانية بـ Render قد تُمسح عند إعادة التشغيل. لمسابقة جادة/طويلة، استخدم قاعدة بيانات خارجية دائمة (مثل Render PostgreSQL المجاني) — أخبرني إذا تبي أطوّر هذا الجزء.
- تأكد إن البوت أدمن فعلي بالقناة وله صلاحية دعوة المستخدمين، وإلا ما راح تنشئ روابط.
- كلمات المرور مشفّرة بقاعدة البيانات (لا تُخزَّن كنص صريح) عبر `werkzeug.security`.
